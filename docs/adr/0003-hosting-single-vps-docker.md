# ADR-0003: 単一VPS(Ubuntu 24.04)+ Docker運用

## ステータス
承認済み(2026-07-06、ユーザーヒアリングにて決定)

## コンテキスト
技術スタックにUbuntu 24.04 LTSとNginxが明記されており、単一VPS運用が前提と見えたため確認したところ、明示的に確定した。

## 検討した選択肢
1. 単一VPS上でDocker運用(採用)
2. クラウドマネージドサービス中心(AWS RDS / ECS 等)
3. VPSから開始し将来クラウド移行を見据える

選択肢2は可用性・スケーラビリティに優れるが、想定規模([[0005-scale-assumption]]: 小規模)に対して運用コスト・複雑性が過剰。

## 決定
単一VPS(Ubuntu 24.04)上でDocker Composeによりサービスを構成する。

## 結果(Consequences)

**メリット**
- コスト最小、指定技術スタックとの整合性が高い
- Docker化により、将来クラウド(ECS/Cloud Run等)へ移行する際もコンテナ単位でそのまま移行しやすい

**デメリット**
- 単一VPSは単一障害点(SPOF)。ディスク障害・VPS消失に対しては外部バックアップで対応する([[../design/01-basic-design.md]] §8 セキュリティ設計)
- 水平スケールはできない(垂直スケール、またはクラウド移行が必要)

**将来の拡張性**
- コンテナ構成のまま、将来的にマネージドDB(RDS等)やオーケストレーション(ECS/K8s)への段階的移行が可能
