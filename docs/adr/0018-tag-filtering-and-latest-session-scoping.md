# ADR-0018: タグによる議案の絞り込み・表示と、ホーム画面の最新会期スコープ化

## ステータス
承認済み(2026-07-07)

## コンテキスト
ユーザーから以下の要望があった。

1. ホーム画面のタグ集計タイルをクリックすると、そのタグの議案一覧が見たい
2. ホーム画面の「議案の状況」に、令和8年2月定例会と6月定例会の議案が合算されて表示されてしまっている。最新の会期だけを表示してほしい
3. 「議案のタグ別件数」も同様に最新の会期だけでよい
4. 議案検索(`/search`)でタグによる絞り込みをできるようにしてほしい
5. 検索結果一覧・議案一覧の両方に、タグのフィルタとタグ自体を表示してほしい

2の原因は、令和8年2月定例会の議案(105件)を検証目的で追加取り込みした([[0017-bill-vote-stance-ingestion]])ことで、ホーム画面が`fetchBills({ limit: 100 })`と会期を指定せずに全件取得していたため、複数会期の議案が合算されるようになったこと。加えてAPIの上限(100件)を超える件数(2会期合計143件)になり、一部の議案が集計から漏れる状態にもなっていた。

## 決定

### 1. タグの解決ロジックを1箇所に集約する(`domain/aiContent/billTags.ts`)
承認済み(is_verified=true)のタグAiContentから、`sourceDocumentId → タグ配列`のマップを組み立てる`buildSourceDocumentTagsMap`と、指定タグを含む議案の`sourceDocumentId`一覧を返す`sourceDocumentIdsForTag`を新設した。既存の`tagCounts.ts`(ホーム画面の集計)もこれに委譲するようリファクタリングし、タグのJSON解析ロジックの重複を避けた。

### 2. `BillWithSourceSchema`に`tags`フィールドを追加し、`BillDetailSchema`の`aiTags`は廃止する
議案一覧・検索結果・議案詳細のすべてで同じ`tags: string[]`フィールドを使うようにした。従来`BillDetailSchema`だけが独自に`aiTags`を持っていたが、意味が重複するため統合した(`buildBillDetail`・議案詳細画面を追従修正)。

### 3. `attachSourceUrl`/`attachSourceUrlToMany`にタグ付与も統合する
原本URLの付与とタグの付与は、どちらも「公開表示用にBillを装飾する」という同じ関心事であるため、別関数に分けず`attachSourceUrl(bill, documentRepository, tagsBySourceDocumentId)`に統合した。呼び出し側(bills一覧・検索・議案詳細)は共通のタグマップを1回だけ組み立てて渡す。

### 4. `BillRepository.findPage`に汎用の`sourceDocumentIds`フィルタを追加する
「タグ」という概念をBillRepositoryに持ち込まず、「事前に解決したsourceDocumentIdの集合で絞り込む」という汎用フィルタとして実装した。タグ→sourceDocumentIdの解決はアプリケーション層(ルートハンドラ)で行う。

### 5. `/api/v1/tags`に`meetingId`クエリパラメータを追加する
指定すると、その会期の議案に紐づくタグのみに絞って集計する。ホーム画面は常に最新会期のmeetingIdを渡す。

### 6. ホーム画面は「最新の会期」を開始日(startDate)で判定する
`Meeting`の一覧取得順(id昇順/降順)は暦日と無関係(UUIDのため)であり、実際に2月定例会追加後は取得順と開催順が一致しない状態になっていた。ホーム画面側で`startDate`の降順ソートを行い、真に最新の会期を明示的に選び直すようにした。議案の状況・タグ別件数の両方をこの会期のmeetingIdでスコープする。

### 7. `/search`のタグ絞り込みはキーワード検索結果に対するAND条件とする
タグのみの絞り込み(キーワードなし)は`/bills?tag=X`で代替できるため、`/search`ではキーワード検索(全文検索)の結果をタグでさらに絞り込む形にした。全文検索エンジン(SQLite FTS5)側にタグ条件を渡すのではなく、ヒットしたBillのタグをアプリケーション層で照合する。

## 結果(Consequences)

**メリット**
- ホーム画面の「議案の状況」「議案のタグ別件数」が、実データ(6月定例会38件・2月定例会105件が混在していた状態)で正しく最新会期のみにスコープされることを確認した。
- タグでの絞り込み(`/bills?tag=X`)、キーワード+タグの複合検索(`/search?q=...&tag=...`)を実データで検証し、期待通りの結果になることを確認した。
- `BillDetailSchema.aiTags`の重複を解消し、タグの表現を1箇所(`tags`)に統一した。

**発見した別件の不具合(本ADRのスコープ外、対応は別途)**
- 全文検索(SQLite FTS5)で、一部のキーワード(例:「市税」)がヒットせず、他のキーワード(例:「専決処分」「第106号」)は正しくヒットするという不整合を実データ検証中に発見した。フレーズクエリのトークナイズに起因すると見られるが、本ADRの対応範囲外のため、今回は修正せず記録のみ残す。

**将来の拡張性**
- `sourceDocumentIds`フィルタは汎用的なため、将来「特定の委員会に付託された議案」等、別の絞り込み軸にも同じ仕組みを再利用できる。
