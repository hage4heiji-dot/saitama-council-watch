# ADR-0021: 検索・クロス集計への期間(会期)フィルタの追加

## ステータス
承認済み(2026-07-07)

## コンテキスト
ユーザーから「検索やクロス集計に期間がない」との指摘があった(docs/adr/0020の要望(1))。議案一覧(`/bills`)には既に`meetingId`によるフィルタがあったが、検索(`/search`)と議員×タグクロス集計(`/analysis`、docs/adr/0019)には期間で絞り込む手段がなかった。

## 決定

### 1. 「期間」の単位は自由な日付範囲ではなく、既存の`Meeting`(会期)とする
市議会の活動は定例会・臨時会という会期単位で構成されており、議案・投票記録もすべて`meetingId`で紐づく。自由な日付ピッカーを追加するのではなく、既存の`Meeting`一覧から選ぶ形にすることで、実装がシンプルになり、かつユーザーにとっても「令和8年2月定例会」のように意味のある単位で絞り込める。

### 2. 検索(`/search`)は`meetingId`クエリパラメータを追加し、議案一覧と同じフィルタ方式にする
`SearchQuerySchema`に`meetingId`を追加し、検索結果を`bill.meetingId`で絞り込む(タグ絞り込みと同じAND条件)。空欄検索(全件表示)の場合は既存の`fetchBills({ meetingId })`にそのまま渡す。

### 3. クロス集計(`/analysis`)は`VoteWithBillInfo`に`billMeetingId`を追加し、`buildLegislatorTagMatrix`にmeetingIdフィルタ引数を追加する
`status`フィルタと同様、`meetingIdFilter`を`buildLegislatorTagMatrix`の第4引数として追加し、投票記録の絞り込みに使う。既存の集計ロジック(全件取得→アプリ側で集計)の方針を踏襲した。

## 結果(Consequences)

**メリット**
- 実データ(令和8年2月定例会・6月定例会)で検証: 検索の`meetingId`絞り込みで会期ごとに異なる件数が返ることを確認。クロス集計は安全なテスト用タグを一時的に付与し、投票記録のある令和8年2月定例会でmeetingId指定時に57行、投票記録のない6月定例会を指定した場合は0行になることを確認した(投票記録自体が2月定例会にしかまだ無いため)。
- 回帰テストを追加(`legislatorTagMatrix.test.ts`): meetingIdFilterで会期ごとに正しく絞り込まれることを検証。

**デメリット・今後の課題**
- 会期単位より細かい日付範囲(例: 特定の1週間)での絞り込みはサポートしない。市議会活動が会期単位で構成されている以上、現時点では不要と判断した。
- クロス集計のさらなるリッチ化(会派ロールアップ、セルからのドリルダウン等)は本ADRの対象外(ユーザー要望の別項目、別途対応)。
