# ADR-0005: 小規模スタート、ただし拡張性を犠牲にしない

## ステータス
承認済み(2026-07-06、ユーザーヒアリングにて決定)

## コンテキスト
インフラ設計・バッチ頻度・キャッシュ戦略のスペック判断の前提として、想定規模を確認した。

## 決定
初期は個人開発・月間数百〜数千PV規模を前提にコストを最小化する。ただし、DB/API/AIパイプラインは疎結合に設計し、将来の規模拡大(中規模: 月間数万PV以上)を妨げない。

## 結果(Consequences)

**このADRが他の判断に与える影響**
- ジョブキュー(BullMQ+Redis)、監視基盤(Prometheus/Grafana)、pgvector等は「今は導入しない(YAGNI)」が「後から追加できる場所は空けておく」([[0001-architecture-style]])
- ページネーションは初期からカーソルベースを採用し、データ増加後の作り直しを避ける([[../design/01-basic-design.md]] §4 API設計)

**将来の拡張性**
- 規模拡大の兆候(レスポンス遅延、VPSリソース逼迫)が見えた時点で、[[0003-hosting-single-vps-docker]] の移行パスに沿ってクラウド化を検討する
